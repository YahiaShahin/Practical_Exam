01

Syntax Basics

C is a compiled language — you write code, a compiler translates it to machine instructions, then it runs. Every statement must end with ;, and every program must have exactly one main() function as the entry point.

Anatomy of a minimal C program
#include <stdio.h> ← ① int main() { ← ② /* your code */ return 0; ← ③ }
What each part does
① Include the standard I/O library so you can use printf and scanf
② main() is where execution starts — every C program has exactly one
③ Return 0 to tell the OS the program succeeded

Comments are ignored by the compiler entirely. They exist for humans reading the code.

// Single-line comment — from // to end of line /* Multi-line comment spans as many lines as you want */ int x = 5; // inline comment after a statement
💡
The semicolon is the most commonly forgotten character in C. Every statement (assignment, function call, declaration) needs one. Braces {} do not need semicolons.

Scopes

Curly braces {} define scopes — isolated regions of code. A variable declared inside a scope only exists inside it. A child scope can see its parent's variables, but a parent cannot reach into a child, and siblings are completely blind to each other.

Scope chain — how visibility works upward

ScopeC (child of B)

sees: a, b, c
Chain goes upward only. ScopeA cannot see b, c, or d. Siblings (C and D) cannot see each other.

⚠
A variable declared inside a scope is destroyed when that scope closes (}). Try to use it outside and the compiler refuses. This is called scope lifetime.

05

Printf & Scanf

printf outputs formatted text to the terminal. scanf reads typed input. Both use format specifiers as typed placeholders that must exactly match the variable type.

Live printf builder — edit values and watch code + output update
Terminal Output
%d→integer (int)
%f→float/double — default 6 decimal places
%.2f→float with exactly 2 decimal places
%c→single char
%s→string (char array)
\n→newline — printf won't add one automatically
// scanf reads INTO a variable's memory address int n; char c; float f; scanf("%d", &n); // & gives the address of n scanf("%f", &f); scanf(" %c", &c); // space before %c skips leftover newline
⚠
Always put a space before %c in scanf (" %c"). Without it, scanf reads the leftover \n from the previous Enter press and your char gets garbage.

08

Functions

A function is a named, reusable block of code. You define it once and call it as many times as you need. Functions take the repetition out of your code and make it easier to read, test, and fix. Every C program already uses one: main().

There are four combinations based on two questions: does the function receive data (parameters)? Does it send data back (return value)?

Type 1
No return  ·  No parameters
void greet(void)
Does a job and finishes. Nothing goes in, nothing comes out. Good for printing menus, drawing separators, or any self-contained action.
Type 2
No return  ·  Has parameters
void printSum(int a, int b)
Receives data and acts on it, but doesn't hand anything back. Good for printing results, modifying output based on input.
Type 3
Has return  ·  No parameters
int getYear(void)
Computes or reads something and hands the result back. No input needed — it figures it out internally. Good for reading sensors, getting constants.
Type 4
Has return  ·  Has parameters
int add(int a, int b)
The most common type. Takes input, processes it, returns a result. Think of it like a math function: f(x) = x + 1.
main.c
Anatomy of a function definition
int add(int a, int b) { ← ① ② ③ return a + b; ← ④ } int main() { int result = add(3, 5); ← ⑤ printf("%d\n", result); // prints 8 return 0; }
What each part does
① Return type — what type of value the function sends back (int, float, void…)
② Name — what you call it. Use a verb: add, print, calculate
③ Parameters — the inputs, each with a type and name. Empty = void
④ return — sends a value back to the caller and exits the function immediately
⑤ Call — use the function name with arguments in parentheses
⚠
A function must be declared before it is called. Either define it above main(), or write a prototype at the top: int add(int a, int b); — this tells the compiler the function exists before it sees the full definition.

1D Arrays

An array is a contiguous block of memory holding multiple values of the same type. Elements are accessed by index starting at 0. An array of size N has valid indices 0 through N−1. Going outside causes undefined behavior — C will not stop you.

Array inspector — click a cell to inspect and edit it
↑ Click any cell to inspect it
Size: arr[0] =
💡
Each int is 4 bytes. So arr[i] lives at memory address base + i * 4. That's why array access is O(1) — the address is computed directly from the index.
⚠
Off-by-one errors are extremely common. A declared array int arr[8] has indices 0–7. Accessing arr[8] is out of bounds — C won't crash immediately, it silently reads or corrupts adjacent memory.

2D Arrays (Matrices)

A 2D array is a grid of values accessed with two indices: mat[row][col]. Internally it's stored as one flat block of memory in row-major order — all of row 0, then all of row 1, and so on. Hover rows/columns to see the highlight; click a cell to inspect it.

The flat memory index formula is: row × numCols + col. For a 4-column matrix, mat[1][2] is at flat index 1×4 + 2 = 6. This is why traversing a 2D array row-by-row is faster than column-by-column — consecutive rows sit next to each other in memory.
